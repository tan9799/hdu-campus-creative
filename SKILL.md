# 杭电印迹 · AI 文创盲盒生成器 v5

## 设计哲学
以“让每一份校园记忆成为可收藏的数字艺术品”为使命，融合盲盒机制、AI 创作、社交分享与**灵感探索**，提供极致个性化体验。v5 版本强化了**多模态理解**、**跨会话持久化**和**社区互动**能力。

## 核心能力矩阵

### 1. 智能场景理解引擎（多模态升级）
- **语义解析**：支持复合句、转折、条件、反问（如“虽然下雨但心情很好，能不能画一个温暖一点的？”）。
- **情感识别**：检测 8 种情绪（兴奋、疲惫、怀旧、宁静、焦虑、期待、自豪、幽默），影响风格权重和文案语调。
- **实体链接**：自动识别杭电地标、活动、课程名称、校园梗，并生成对应的视觉元素强制提示。
- **多模态输入**（新增）：用户可上传一张参考图（或描述图片内容），引擎提取主色调、构图、风格特征，用于影响生成方向。输出时标注“灵感来源于您的参考图”。
- **负面输入处理**：若用户表达不满（如“太暗了”），自动调整亮度、对比度，并记录偏好到用户画像（降权该风格或该参数）。

### 2. 跨会话用户画像（冗余存储 + 在线学习）
- **持久化存储**：优先使用 Matrix `account_data`，失败时降级至本地 IndexedDB（Web）或 SQLite（CLI），并提醒“本次进度可能无法跨设备同步”。
  - 存储结构：
    ```json
    {
      "collected_styles": ["复古胶片风", "赛博朋克风"],
      "preferences": { "复古胶片风": 0.8, "水墨国风": -0.2 },
      "challenge_achievements": ["首次生成", "风格收集者"],
      "recent_generations": ["id1", "id2"],   // 最多20条
      "negative_feedback": ["id3"],            // 踩过的作品ID
      "last_active": "2026-05-24T10:00:00Z"
    }
动态权重学习：每次用户反馈（赞/踩）后，调整对应风格权重：

赞：权重 × 1.2（最高 2.0）

踩：权重 × 0.5（最低 0.1）

未收集风格基础权重 = 1.0，已收集风格基础权重 = 0.3

多设备合并策略：若检测到多个设备存储差异，以最新 last_active 为准，同时保留所有已收集风格并集。

3. 渐进式图像加载（预测式预加载）
首屏响应（≤200ms）：

返回文案 caption

返回风格化占位图：根据选中的风格，动态生成一个 base64 编码的 SVG 色块（如赛博朋克风为深蓝紫色渐变），并叠加半透明“HDU”字样。

返回 generationId 和 estimatedTime（预计剩余秒数）

异步生成：后台调用通义万相 API，采用指数退避重试（2s, 4s, 8s，最多3次）。成功后通过 Matrix m.replace 更新原消息；若平台不支持，则发送一条新消息并引用原占位消息。

智能降级：第3次失败后，从“杭电精选实景图库”（预置10张授权图片）中随机选择一张，应用用户请求的风格滤镜（通过 CSS filter 或 canvas 简单处理），并告知用户“已为您切换至实景应急图”。

4. 社交收藏与分享（灵感集市）
作品 ID：hdu_{timestamp}_{random6}，支持点赞、收藏、举报。

作品画廊：/gallery 展示用户历史作品（缩略图 + 风格 + 点赞数），支持分页和删除。

热门榜单：/hot 展示近7天点赞数前10的作品（模拟数据，但可通过存储实现真实计数）。

灵感集市（新增）：用户可通过 /inspire 浏览他人公开发布的作品（随机20个），点击“试试同款”自动填充描述和风格，生成新作品（需用户确认）。

分享链接：生成短链 https://skill.mzsjai.com/s/{id}，打开后展示高清图、文案、生成参数，并包含一键复刻按钮。

5. 多模式与彩蛋系统（扩展）
已有彩蛋保留：评委辛苦了、毕业季、挑战达人、隐藏款（集齐4风格后解锁“杭电之光”特殊画框）。

新增彩蛋：

输入 /theme 节日 切换主题（校庆版、新年版、毕业季版），改变 UI 配色和默认文案风格。

输入 /duel @对手 发起风格对战（两人各生成一张，AI 模拟投票决胜，输出“胜出风格”及理由）。

输入 /timecapsule 生成一张“未来明信片”：用户输入未来的日期（如2028年毕业时），AI 生成该时间点的校园想象图（基于当前场景 + 岁月痕迹）。

隐藏彩蛋：连续生成3张同风格 → 触发“风格大师”徽章，并解锁该风格的专属边框。

执行流程（含高级特性）
Step 0: 用户画像加载与合并
根据发送者 Matrix ID 加载所有可用存储源的数据，合并后更新到最新存储。

Step 1: 消息解析
支持短命令（见 references/commands.md）。

非命令消息：

调用口语清洗（references/input-cleaning.md）

情感分析（调用本地模型或云 API）

实体识别（杭电地标/梗）

多图数量提取（如“来三张” → N=3）

Step 2: 风格选择（加权随机 + 互异算法）
当 N=1 时：

python
weight = base_weight * preference_factor * emotion_factor
# base_weight: 未收集1.0，已收集0.3
# preference_factor: 喜欢+0.5，不喜欢-0.8（截断至0.1）
# emotion_factor: 兴奋→赛博朋克×1.5，宁静→水墨国风×1.5，等等
当 N>1 时（多图模式）：

优先从未收集风格中不重复选取；若不足，则从全体风格中按“历史出现次数最少优先”选取。

最终列表随机打乱顺序。

Step 3: 提示词构建
模板：[主体] + [强制杭电元素] + [光线/天气] + [色彩倾向] + [风格关键词] + [情感修饰] + [水印约束]

调用 references/prompt-optimization.md 中的风格专属关键词库。

强制添加负面提示词：no text, no watermark, no low quality, no deformed hands, no extra limbs。

Step 4: 图像生成（异步 + 预测进度）
立即返回占位响应，启动后台任务。

后台任务使用指数退避重试，并实时推送进度（25% 调用中，50% 生成中，75% 渲染中，100% 完成）。

若最终失败，切换应急图库并记录错误日志。

Step 5: 文案生成（同步）
基于情感标签和风格，从预置模板库中组合或调用大模型生成。

模板示例：

幽默：“bug 退散，杭电保佑！”（赛博朋克 + 兴奋）

文艺：“月雅湖的晨雾里，藏着代码的诗。”（水墨 + 宁静）

励志：“实验室的灯，照亮未来的路。”（复古 + 怀旧）

Step 6: 进度与成就
更新用户画像：collected_styles 新增未收集的风格。

检测成就条件：

首次生成 → 🏅 杭电新生

集齐4种风格 → 🎉 文创大师（解锁隐藏款）

连续3次点赞 → 👍 品味之选

使用 /duel 并获胜 → ⚔️ 风格赢家

输出动态进度条，使用 emoji 或 Unicode 字符。

Step 7: 作品存储与发布
存储作品记录：{ id, style, caption, imageUrl, likes, creatorId, timestamp, public }

自动设为公开（用户可通过 /privacy 设为私密）。

若用户允许，将作品加入“灵感集市”公共池。

Step 8: 用户反馈闭环
每次生成后，在回复消息底部附加：

text
👍 满意？回复“赞”或“👍” | 👎 不满意？回复“踩”或“👎”
您的反馈将优化后续推荐 ✨
反馈处理：

赞：generationId 标记正面，调整风格权重（+0.2）

踩：标记负面，调整风格权重（-0.3），并询问“是否要换一种风格重新生成？”若用户回复“是”，则自动执行 /retry 并排除当前风格。

错误处理与降级
API 完全失败 → 使用应急图库 + 提示用户可 /retry

用户画像存储失败 → 会话级内存存储 + 警告“进度不会跨设备保存”

命令不可识别 → 模糊匹配最相似命令（编辑距离 ≤ 2），并提示“您是否想输入 /xxx？”

多图生成部分失败 → 返回成功部分 + 列出失败索引，允许 /retry 2,3 单独重试

扩展性说明
风格配置：修改 references/style-guide.md 即可新增风格，无需改动核心逻辑。

彩蛋系统：在 references/easter-eggs.md（新增）中定义触发条件和响应。

Webhook 预留：支持与校园实体活动联动（如生成图可兑换文创周边），回调地址可在环境变量配置。

依赖与限制
Matrix 平台能力：

需要 account_data 支持（用于跨会话持久化）

需要 m.replace 或消息编辑支持（用于异步图像更新）

若平台不支持，自动降级为同步模式（等待图像生成完成再回复）

图像生成 API：通义万相或兼容 OpenAI 格式的接口，需配置 API Key。

应急图库：预置10张杭电实景图（已获授权），存储于 /assets/fallback/。
